## 1. 首页导航（Guid 页）

**描述**：首页是应用的核心入口，用户可以在这里选择 AI Agent、输入消息发起新对话，并通过侧边栏快速切换对话和管理 Agent。

**用户操作路径**：
1. 启动应用后进入 Guid 首页
2. 点击顶部 Agent 选择器，从下拉列表切换当前使用的 AI Agent
3. 在输入框中输入消息
4. 点击发送按钮或按 Enter 键发起新对话，自动跳转到对话页面
5. 通过左侧侧边栏切换"消息"和"Agent"两个标签页
6. 点击输入框旁的"+"按钮，选择"上传文件"或"关联文件夹对话"
7. 点击快速启动卡片，自动切换至对应 Agent 并预填提示词

- [ ] AC1：Guid 页面正常加载，页面有实质性内容
- [ ] AC2：页面顶部有 Agent 选择器，显示当前选中的 Agent 名称（非空）
- [ ] AC3：Agent 选择器带有可展开的下拉图标（chevron），点击后弹出 Agent 切换列表
- [ ] AC4：从下拉列表选择另一个 Agent 后，选择器显示的名称更新为所选 Agent
- [ ] AC5：页面中部有文本输入框（位于 Agent 选择器下方），可以输入文字
- [ ] AC6：侧边栏顶部有"消息"和"Agent"两个标签，均可见且可点击

## 2. 对话核心流程

**描述**：用户与 AI Agent 进行实时对话，消息发送后在对话页面中看到消息历史和 AI 回复。页面采用三栏布局：顶部标题栏、中间消息列表（虚拟滚动）、底部 SendBox；右侧可选 Workspace 和预览面板。

**用户操作路径**：
1. 在首页或对话页选择 Agent 和模型
2. 在输入框输入消息（超过 800 字符自动切换多行模式）
3. 点击发送或按 Enter 发送；Shift+Enter 换行
4. Agent 回复时流式展示内容；思考过程可展开/折叠并显示耗时
5. Agent 调用工具时，消息列表展示工具调用卡片（含状态和 Diff 预览）
6. 部分工具调用需要用户确认（Allow Once / Allow Always / Deny）
7. 继续在同一对话中追问
8. 双击标题栏会话名称可直接重命名
9. 点击标题栏中间的搜索按钮，打开当前会话内消息搜索，输入关键词定位消息
10. 点击"历史"按钮查看该 Agent 的历史会话，可切换或发起新会话；每行有删除按钮，点击后弹窗确认删除
11. 消息列表向上滚动时出现"回到底部"按钮，点击跳回最新消息

- [ ] AC3c：标题栏右侧有历史按钮（时钟图标）和定时任务徽章（闹钟图标）

## 3. 对话管理操作

**描述**：用户可以对侧边栏中的对话进行搜索、重命名、删除和置顶等管理操作。侧边栏搜索为全局跨会话搜索，覆盖所有历史消息内容。

**用户操作路径**：
1. **全局搜索**：点击侧边栏搜索触发器或按 `Cmd/Ctrl+Shift+F`，打开全局搜索弹窗，在所有历史对话的消息内容中检索
2. **重命名**：悬停某条对话，点击出现的三点菜单，选择"重命名"，在弹窗中修改名称后确认
3. **删除**：悬停某条对话，点击三点菜单，选择"删除"，在确认弹窗中点击确认
4. **置顶**：悬停某条对话，点击三点菜单，选择"置顶/取消置顶"

**验收标准（AC）**：
- [ ] AC1：侧边栏中有搜索触发器按钮，点击或按 `Cmd/Ctrl+Shift+F` 后打开全局搜索弹窗
- [ ] AC2：全局搜索弹窗包含文本输入框，输入关键词后 250ms 防抖查询数据库（SQLite LIKE 匹配）
- [ ] AC3：搜索结果以列表展示，每条结果显示：会话 Agent 头像、会话名称、时间戳、消息内容预览（最多 110 字符，关键词高亮）
- [ ] AC4：结果列表支持滚动分页加载（每页 20 条）；点击某条结果跳转到对应会话并定位到目标消息
- [ ] AC5：弹窗显示最近搜索历史（最多 8 条，存储于 localStorage）；无输入时展示历史关键词
- [ ] AC6：按 Escape 键或点击弹窗外关闭搜索弹窗
- [ ] AC7：悬停对话行后，出现三点菜单触发按钮
- [ ] AC8：点击三点菜单后，下拉菜单包含"重命名"选项
- [ ] AC9：点击三点菜单后，下拉菜单包含"删除"选项
- [ ] AC10：点击三点菜单后，下拉菜单包含"置顶/取消置顶"选项
- [ ] AC10a：点击"置顶"后，该对话移动到侧边栏列表顶部，并有置顶标识
- [ ] AC10b：再次点击"取消置顶"后，该对话从置顶区域移回普通列表，置顶标识消失
- [ ] AC11：点击"重命名"后，弹出包含文本输入框的模态框，输入框可编辑
- [ ] AC12：重命名弹窗中有"取消"和"确认"按钮，点击取消关闭弹窗
- [ ] AC13：点击"删除"后，弹出包含"取消"和"确认"按钮的确认框
- [ ] AC14：删除确认框点击"取消"后，弹窗关闭，对话未被删除

**优先级**：P1

**状态**：已实现

## 4. 消息输入框（SendBox）

**描述**：对话页面底部的输入区域，支持文本输入、文件附件、模型选择等功能，并提供发送和停止按钮。

**用户操作路径**：
1. 在对话页面底部输入框中输入文字
2. 点击发送按钮或按 Enter 发送消息
3. Agent 处理中时，发送按钮变为停止按钮，点击可中断生成
4. 点击设置图标（齿轮）打开设置弹窗，可切换模型、权限模式等参数（参数项因 Agent 类型动态变化）
5. 使用 `@` 符号触发文件引用菜单，选择工作区文件作为上下文
6. 使用 `/` 符号触发斜杠命令菜单
7. 拖拽文件到输入框以附加文件
8. 粘贴图片或文件到输入框

**验收标准（AC）**：
- [ ] AC1：对话页面底部有 SendBox 面板，包含文本输入区域
- [ ] AC2：文本输入框可编辑，支持输入文字
- [ ] AC3：输入框为空时，发送按钮处于禁用状态
- [ ] AC4：输入框有内容时，发送按钮变为启用状态
- [ ] AC5：Agent 空闲状态下，停止按钮不可见
- [ ] AC6：SendBox 底部工具栏可见
- [ ] AC7：设置图标（齿轮按钮）在 SendBox 中可见
- [ ] AC8：点击设置图标，弹出设置浮层，浮层至少包含一个参数行（不同 Agent 显示的参数项不同，空节点自动隐藏）

  - Aionrs：default / auto_edit / yolo
- [ ] AC11：Codex 类型 Agent 的设置浮层额外包含"配置"参数行，展示从后端动态拉取的 select 类型配置项
- [ ] AC12：设置浮层完全在视口范围内，不被父容器裁剪（回归：image #27 bug）
- [ ] AC13：设置浮层打开时，页面存在点击外部关闭的透明遮罩层
- [ ] AC14：点击浮层外部区域，设置浮层关闭
- [ ] AC15：反复打开关闭设置浮层 3 次，不产生严重的控制台错误
- [ ] AC16：输入 `@` 字符时，触发文件引用下拉菜单
- [ ] AC17：输入 `/` 字符时，触发斜杠命令下拉菜单

**优先级**：P0

**状态**：已实现（AC14、AC15 为部分实现，依赖工作区文件）

## 5. 工作区面板（Workspace）

**描述**：对话页面右侧的工作区面板，展示当前对话关联的文件目录，Agent 可在此读写文件，用户可浏览文件树、查看 Git 变更、预览文件。工作区按对话维度绑定，每个对话有独立的工作区路径。

**用户操作路径**：
1. 打开一个已配置工作区的对话，右侧面板自动展示文件目录树
2. 在文件树中浏览、搜索文件；单击文件在预览面板中打开
3. 双击文件，或右键选择"添加至聊天"，将文件注入当前对话输入框
4. 右键菜单支持打开、预览、下载、在系统中显示、删除、重命名文件
5. 切换到"变更"标签，查看 Git 暂存/未暂存文件，可逐文件或批量暂存
6. 拖拽文件/文件夹到工作区面板，导入到当前工作区目录
7. 粘贴文件到工作区，弹出确认框后导入（可设置"不再询问"）
8. 临时工作区可通过迁移功能复制到永久目录，同步创建新会话

**验收标准（AC）**：
- [ ] AC1：对话页面的工作区面板在 DOM 中存在（即使折叠状态）
- [ ] AC2：未关联工作区目录时，面板显示空状态提示（含标题和描述文字）
- [ ] AC3：工作区面板顶部有"文件"和"变更"两个主标签，可切换
- [ ] AC4："文件"标签下展示文件目录树，根节点显示关联的文件夹名称，支持文件夹懒加载展开/折叠
- [ ] AC4a：文件树包含隐藏文件/文件夹（以 `.` 开头的 dotfiles）
- [ ] AC4b：面板有刷新按钮，点击后重新扫描工作区目录

6. 在其他设备浏览器中访问，看到登录页，输入账号/密码后进入主应用

## 18. 扩展系统（Extensions）

**描述**：AionUi 支持加载第三方扩展（manifest 文件：`aion-extension.json`），扩展可以贡献 ACP 适配器、助手、Agent 预设、技能、MCP 服务器、渠道插件、WebUI 路由、CSS 主题、设置标签页、模型提供商。扩展有权限系统（safe/moderate/dangerous），支持运行时启用/禁用。

**用户操作路径**：
1. 将扩展目录配置到环境变量 `AIONUI_EXTENSIONS_PATH`（多路径用系统路径分隔符分隔）或放到 `~/.aionui/extensions` 目录
2. 应用启动时自动发现并加载扩展（不可加载：跳过并记录错误）
3. 在设置侧边栏查看扩展贡献的额外设置标签页（可精确定位到某个内置标签页的前后）
4. 通过 IPC 接口查询已加载的扩展列表及其贡献内容
5. 通过 IPC 禁用/启用某个扩展（状态持久化）

**验收标准（AC）**：

**— 扩展发现与加载 —**
- [ ] AC1：`AIONUI_EXTENSIONS_PATH` 配置后，扩展目录被自动扫描（扫描 `aion-extension.json` 清单文件）
- [ ] AC2：有效的扩展 manifest 可被加载（Zod 验证通过）；无效 manifest 不崩溃应用，仅跳过
- [ ] AC3：扩展名称规则：kebab-case，2–64 字符，不可使用 `builtin-` / `aionui-` 等保留前缀

**— 扩展贡献内容 —**
- [ ] AC4：扩展贡献的 ACP 适配器（Agent）可通过 IPC（`extensions.getAcpAdapters`）查询到
- [ ] AC5：扩展贡献的助手（Assistant）可通过 IPC（`extensions.getAssistants`）查询到
- [ ] AC6：扩展贡献的 MCP 服务器可通过 IPC（`extensions.getMcpServers`）查询到，在 MCP 设置页以只读形式展示
- [ ] AC7：扩展贡献的渠道插件（Channel Plugin）出现在 WebUI Channels 设置列表，支持动态字段表单
- [ ] AC8：扩展贡献的设置标签页出现在设置侧边栏，可通过 `position.anchor + placement` 精确定位（在指定内置标签页前/后插入）
- [ ] AC9：扩展贡献的 CSS 主题出现在显示设置的主题列表中

## 19. Agent 市场（Hub）

**描述**：用户可以通过内置的 Agent 市场发现和安装新的 AI Agent CLI 工具，安装后 Agent 自动出现在检测列表和首页 Agent 选择器中。安装状态实时通过 IPC 事件推送，安装完成后 SWR 缓存自动刷新。

**用户操作路径**：
1. 进入设置 → Agent 设置 → 本地 Agent 页面，点击"从市场安装（Install from Market）"按钮
2. Hub 弹窗展示可用 Agent 卡片网格，每张卡片显示图标、名称、描述和安装状态按钮
3. 点击某 Agent 的"Install"按钮，卡片状态变为"Installing..."（loading），等待安装完成
4. 安装完成后，状态变为"Installed"（禁用按钮）
5. 关闭弹窗后，新安装的 Agent 出现在已检测 Agent 列表中
6. 返回首页，在 Agent 选择器中看到新 Agent

**验收标准（AC）**：

- [ ] AC4：卡片显示：Agent 图标、显示名称（`displayName`）、描述
- [ ] AC5：每张卡片有安装状态按钮，状态对应：not_installed（"Install"）/ installing（"Installing..."，禁用）/ installed（"Installed"，禁用灰色）/ install_failed（"Retry"，带错误 Tooltip）/ update_available（"Update"）
