## 1. 首页导航（Guid 页）

**描述**：首页是应用的核心入口，用户可以在这里选择 AI Agent、输入消息发起新对话，并通过侧边栏快速切换对话和管理 Agent。

**用户操作路径**：
1. 登录后进入 Guid 首页
2. 通过顶部 Agent 选择栏选择当前使用的 AI Agent
3. 在输入框中输入消息
4. 点击发送按钮或按 Enter 键发起新对话，自动跳转到对话页面
5. 通过左侧侧边栏切换"消息"和"Agent"两个标签页

**验收标准（AC）**：
- [ ] AC1：Guid 页面正常加载，页面有实质性内容
- [ ] AC2：Agent 选择栏（Pill Bar）可见，至少展示一个 Agent 选项
- [ ] AC3：每个 Agent 选项显示 Agent 名称，并带有可展开的下拉图标
- [ ] AC4：点击某个 Agent 选项后，该选项变为选中状态（`data-agent-selected=true`）
- [ ] AC5：页面底部有文本输入框，可以输入文字
- [ ] AC6：侧边栏顶部有"消息"和"Agent"两个标签，均可见且可点击
- [ ] AC7：点击"Agent"标签后，侧边栏切换为 Agent 列表（有分组标题）
- [ ] AC8：点击"消息"标签后，切换回对话列表
- [ ] AC9：首页展示助手选择区域，用户可以看到可用助手列表

**优先级**：P0

**状态**：已实现

## 2. 对话核心流程

**描述**：用户与 AI Agent 进行实时对话，消息发送后在对话页面中看到消息历史和 AI 回复。

**用户操作路径**：
1. 在首页或对话页选择 Agent 和模型
2. 在输入框输入消息
3. 点击发送或按 Enter，消息发送给 Agent
4. 等待 Agent 回复，实时展示回复内容（流式输出）
5. 继续在同一对话中追问
6. 点击对话标题栏中的"历史"按钮，可查看该 Agent 的历史会话
7. 从历史面板中点击某条历史会话，切换到对应会话

**验收标准（AC）**：
- [ ] AC1：发送消息后，页面跳转到 `/conversation/:id` 对话页面
- [ ] AC2：对话页面顶部标题栏显示非空的会话标题
- [ ] AC3：对话页面底部有文本输入框，可输入文字
- [ ] AC4：消息列表至少渲染一条消息项（`.message-item`）
- [ ] AC5：每条消息显示头像/名称头部区域（首条消息出现，相同发送方连续消息不重复显示）
- [ ] AC6：对话页面标题栏中有"历史"按钮，点击后展开历史下拉面板
- [ ] AC7：历史下拉面板中包含"新会话"选项
- [ ] AC8：历史下拉面板中列出该 Agent 的历史会话，当前会话高亮显示
- [ ] AC9：点击历史面板中的其他会话，成功跳转到对应会话页面
- [ ] AC10：点击历史面板中的"新会话"，创建新会话并跳转，URL 变更
- [ ] AC11：点击历史面板外部区域，面板自动关闭
- [ ] AC12：ACP 类型 Agent（Claude Code、Codex 等）发送消息后，状态徽章变为 `session_active`

**优先级**：P0

**状态**：已实现（ACP 后端测试依赖 CLI 工具安装）

## 3. 对话管理操作

**描述**：用户可以对侧边栏中的对话进行搜索、重命名、删除和置顶等管理操作。

**用户操作路径**：
1. **搜索**：点击侧边栏中的搜索触发器，输入关键词过滤对话列表
2. **重命名**：悬停某条对话，点击出现的三点菜单，选择"重命名"，在弹窗中修改名称后确认
3. **删除**：悬停某条对话，点击三点菜单，选择"删除"，在确认弹窗中点击确认
4. **置顶**：悬停某条对话，点击三点菜单，选择"置顶/取消置顶"

**验收标准（AC）**：
- [ ] AC1：侧边栏中有搜索触发器按钮，点击后打开搜索弹窗
- [ ] AC2：搜索弹窗包含文本输入框，可输入关键词
- [ ] AC3：输入关键词后（经去抖处理），搜索弹窗仍保持可见状态
- [ ] AC4：按 Escape 键关闭搜索弹窗
- [ ] AC5：悬停对话行后，出现三点菜单触发按钮
- [ ] AC6：点击三点菜单后，下拉菜单包含"重命名"选项
- [ ] AC7：点击三点菜单后，下拉菜单包含"删除"选项
- [ ] AC8：点击三点菜单后，下拉菜单包含"置顶/取消置顶"选项
- [ ] AC9：点击"重命名"后，弹出包含文本输入框的模态框，输入框可编辑
- [ ] AC10：重命名弹窗中有"取消"和"确认"按钮，点击取消关闭弹窗
- [ ] AC11：点击"删除"后，弹出包含"取消"和"确认"按钮的确认框
- [ ] AC12：删除确认框点击"取消"后，弹窗关闭，对话未被删除

**优先级**：P1

**状态**：已实现

## 4. 消息输入框（SendBox）

**描述**：对话页面底部的输入区域，支持文本输入、文件附件、模型选择等功能，并提供发送和停止按钮。

**用户操作路径**：
1. 在对话页面底部输入框中输入文字
2. 点击发送按钮或按 Enter 发送消息
3. Agent 处理中时，发送按钮变为停止按钮，点击可中断生成
4. 点击设置图标（齿轮）打开设置弹窗，可切换模型等参数
5. 使用 `@` 符号触发文件引用菜单，选择工作区文件作为上下文
6. 使用 `/` 符号触发斜杠命令菜单
7. 拖拽文件到输入框以附加文件
8. 粘贴图片或文件到输入框

**验收标准（AC）**：
- [ ] AC1：对话页面底部有 SendBox 面板，包含文本输入区域
- [ ] AC2：文本输入框可编辑，支持输入文字
- [ ] AC3：输入框为空时，发送按钮处于禁用状态
- [ ] AC4：输入框有内容时，发送按钮变为启用状态
- [ ] AC5：Agent 空闲状态下，停止按钮不可见
- [ ] AC6：SendBox 底部工具栏可见
- [ ] AC7：设置图标（齿轮按钮）在 SendBox 中可见
- [ ] AC8：点击设置图标，弹出设置浮层，浮层包含至少一个标签行
- [ ] AC9：设置浮层显示"模型"标签，可在此切换模型
- [ ] AC10：设置浮层完全在视口范围内，不被父容器裁剪（回归：image #27 bug）
- [ ] AC11：设置浮层打开时，页面存在点击外部关闭的透明遮罩层
- [ ] AC12：点击浮层外部区域，设置浮层关闭
- [ ] AC13：反复打开关闭设置浮层 3 次，不产生严重的控制台错误
- [ ] AC14：输入 `@` 字符时，触发文件引用下拉菜单
- [ ] AC15：输入 `/` 字符时，触发斜杠命令下拉菜单

## 5. 工作区面板（Workspace）

**描述**：对话页面右侧的工作区面板，展示当前对话关联的文件目录，Agent 可在此读写文件，用户可浏览文件树。

**用户操作路径**：
1. 打开一个已配置工作区的对话
2. 右侧工作区面板展示文件目录树
3. 用户可在工作区面板中浏览文件、通过上下文菜单操作文件
4. 工作区面板支持折叠/展开
5. Agent 修改文件时，文件变更在面板中实时更新

**验收标准（AC）**：
- [ ] AC1：对话页面的工作区面板在 DOM 中存在（即使折叠状态）
- [ ] AC2：已启用工作区的对话，工作区面板展示可见内容，内容非空
- [ ] AC3：工作区面板有标签页栏（WorkspaceTabBar），支持多文件标签
- [ ] AC4：工作区支持右键上下文菜单（新建、重命名、删除文件等操作）
- [ ] AC5：工作区文件夹选择功能可用，可通过设置指定工作区路径
- [ ] AC6：工作区面板有工具栏，包含常用操作按钮

**优先级**：P1

**状态**：已实现（E2E 验证了 DOM 存在，详细功能为部分覆盖）

## 6. 预览面板（Preview）

**描述**：可即时预览 AI 生成的文件，支持 10 余种格式（PDF、Word、Excel、PPT、Markdown、代码、图片、HTML 等），无需切换外部应用。

**用户操作路径**：
1. Agent 生成或修改文件后，预览面板自动展示文件内容
2. 用户可在预览面板中切换已打开的多个文件标签
3. 支持查看文件版本历史（基于 Git），并可回滚到历史版本
4. Markdown 和代码文件支持在预览面板内直接编辑
5. 实时追踪文件变更，Agent 修改文件后预览自动刷新

**验收标准（AC）**：
- [ ] AC1：PDF 文件可在预览面板中渲染显示
- [ ] AC2：Markdown 文件可在预览面板中渲染，支持实时预览
- [ ] AC3：代码文件（JS、TS、Python 等）可在预览面板中展示，有语法高亮
- [ ] AC4：图片文件（PNG、JPG、SVG 等）可在预览面板中展示
- [ ] AC5：HTML 文件可在预览面板中渲染
- [ ] AC6：Word/Excel/PPT 文件有对应的预览支持 [待确认：是否依赖外部库]
- [ ] AC7：多文件同时打开时，面板顶部有标签页，可切换
- [ ] AC8：版本历史功能可查看文件历史记录并回滚（依赖 Git）

## 7. Agent 管理

**描述**：用户可以管理本地 CLI Agent（如 Claude Code、Codex、Gemini 等）和远程 Agent，配置每个 Agent 的默认模型和权限模式。

**用户操作路径**：
1. **查看 Agent**：进入设置 → Agent 设置页，查看已检测到的本地 Agent 和远程 Agent
2. **配置本地 Agent**：点击某个本地 Agent，进入详情页，设置默认模型和权限模式
3. **配置远程 Agent**：点击某个远程 Agent，进入详情页，填写 API URL 和认证信息后保存
4. **侧边栏 Agent 列表**：在主界面左侧边栏切换到"Agent"标签，查看按分组的 Agent 列表
5. **从 Agent 发起对话**：点击侧边栏中某个 Agent 行，跳转到对应 Agent 的对话页
6. **安装新 Agent**：在本地 Agent 设置页点击"从市场安装"按钮

- [ ] AC2：本地 Agent 详情页（如 Claude）显示 Agent 名称

- [ ] AC4：本地 Agent 详情页有"权限模式"下拉选择器，点击后展开选项
- [ ] AC5：Gemini 本地 Agent 详情页正常加载，显示 Gemini 名称
- [ ] AC6：远程 Agent 详情页有名称输入框、URL 输入框（`placeholder="https://"`）和认证类型选择器

## 8. 助手（Assistant）管理

**描述**：用户可以浏览、创建和编辑自定义助手（Assistant），每个助手有独立的系统提示词和技能配置。应用内置 12 个专业助手。

**用户操作路径**：
1. **浏览助手**：进入 `/assistants` 页面，查看所有助手（内置 + 自定义），支持按分类筛选
2. **发起对话**：点击助手卡片上的"Chat"按钮，跳转到首页并带上助手参数
3. **编辑助手**：点击助手卡片上的"Edit"按钮，进入助手详情编辑页
4. **创建助手**：
   - 方式一：在 `/assistants` 页面点击新建按钮
   - 方式二：在侧边栏"助手"分组标题旁点击"+"按钮
   - 填写名称、描述、系统提示词，配置技能，保存
5. **管理技能**：在助手详情页，通过折叠面板查看并启用/禁用该助手的技能

**验收标准（AC）**：
- [ ] AC1：`/assistants` 页面成功加载，页面有实质性内容
- [ ] AC2：`/assistants` 页面有搜索输入框

## 9. 团队模式（Team）

**描述**：用户可以创建包含多个 AI Agent 的团队，指定一个领导 Agent 协调任务，支持多 Agent 并行协作完成复杂任务。

**用户操作路径**：
1. **创建团队**：在侧边栏"团队"分组旁点击"+"按钮，弹出创建模态框
2. 在模态框中填写团队名称，从下拉列表选择调度 Agent（仅支持白名单内的 Agent 类型）
3. 点击"创建"，成功后跳转到团队页面（`/team/:id`）
4. **与领导 Agent 对话**：在团队页面底部输入框输入任务指令，发送后由领导 Agent 处理
5. **动态添加成员**：通过对话指令让领导 Agent 添加其他类型的 Agent 作为团队成员
6. **解散成员**：通过对话指令让领导 Agent 解散某个成员
7. 团队页面顶部有 Agent 标签页，切换可查看各个 Agent 的独立对话

**验收标准（AC）**：
- [ ] AC1：侧边栏有"团队（Teams）"分组区域，旁边有"+"创建按钮
- [ ] AC2：点击"+"打开创建团队模态框，标题为"创建团队（Create Team）"
- [ ] AC3：模态框包含团队名称输入框和调度 Agent 下拉选择器
- [ ] AC4：未填写名称或未选择 Agent 时，确认按钮处于禁用状态
- [ ] AC5：创建团队下拉列表只展示白名单内的 Agent 类型（Claude Code、Codex、Gemini 等）
- [ ] AC6：表单填写完整后，确认按钮变为启用状态，点击后跳转到 `/team/:id`
- [ ] AC7：团队侧边栏中显示新建的团队名称
- [ ] AC8：团队页面有底部聊天输入框，可输入并发送消息给领导 Agent
- [ ] AC9：发送的消息成功在对话区域展示
- [ ] AC10：领导 Agent 可通过对话指令动态添加新成员，成员标签页出现在团队页面顶部 [待确认：E2E 中有超时风险]
