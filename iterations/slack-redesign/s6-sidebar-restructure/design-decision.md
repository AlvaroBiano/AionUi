# S6: 侧边栏结构重组

**Date**: 2026-03-30
**Status**: 设计讨论中
**Trigger**: 用户验收后发现整体概念仍有混乱，发起产品层面的重新审视

---

## 问题诊断

### 1. App 身份模糊
同时在做三件事：Slack 团队协作（频道）、ChatGPT 式对话（选模型聊天）、Agent 平台（助手市场）。用户打开不知从何开始。

### 2. 同一件事太多入口
"发起对话"至少 5 种方式：+ 新会话、通用 Agent 旁 +、Agent 选择器横条、助手快捷标签、侧边栏 Agent 名称。

### 3. 侧边栏按技术类型分组，不直觉
"频道 / 通用 Agent / 助手"三个 section — 普通用户无法理解"通用 Agent"和"助手"的区别。

### 4. 主区域信息层级平坦
问候语、Agent 选择条、输入框、助手标签云四个模块等权展示，无主次。

---

## 核心设计决策

### 决策 1: 侧边栏从"按技术类型分组"改为"按交互形式分组"

```
现在：频道 / 通用Agent / 助手      （按技术架构分）
改为：收藏 / 频道 / 私信            （按交互形式分）
```

**理由**：用户关心"这个助手能帮我做什么"，不关心"它底层是 CLI 还是 preset"。技术层级关系（助手基于通用 Agent 搭建）在 AgentProfileSider 详情页展示，不在侧边栏暴露。

### 决策 2: 私信列表只显示聊过的助手

```
现在：所有已配置的 agent 都平铺在侧边栏
改为：只显示有过对话的 agent，按最近活跃时间排序
```

**理由**：参考 Slack — 公司 500 人，侧边栏只显示聊过的人。没聊过的通过搜索或"+ 新对话"发现。

**机制**：
- 按最近活跃排序，常用自然浮上来
- 不活跃的自动折叠到"显示更多"
- ⭐ 收藏钉在顶部，不受排序影响
- 🔍 搜索快速定位

### 决策 3: 点击 Agent 展开对话列表（非直接进入最近对话）

**理由**：需要兼容"一个助手多条对话"的场景。用户可能同时跟同一个助手处理不同任务。

### 决策 4: 合并重复入口

```
"+ 新对话"（顶部）和"+ 发起私信"（私信旁）做的是同一件事 → 去掉后者
频道旁 "+" 是创建群聊，独立功能 → 保留
```

**最终入口**：
- `+ 新对话` → 导航到首页（GuidPage），选助手开始聊
- 频道旁 `+` → 创建群聊

## 与现状对比

| 维度 | 现在 | S6 方案 |
|------|------|---------|
| Section 分组 | 频道 / 通用Agent / 助手 | 收藏 / 频道 / 私信 |
| 分组依据 | 技术类型 | 交互形式 |
| 私信列表 | 所有已配置 agent | 只显示聊过的 |
| 排序 | 无明确排序 | 按最近活跃时间 |
| 数量控制 | 全部平铺 | 自动折叠 + "显示更多" |
| 类型区分位置 | 侧边栏 section 标题 | AgentProfileSider 详情页 |
| 发现新助手 | 侧边栏直接可见 | `+ 新对话` → 首页 |
| 新对话入口 | 5 个 | 2 个（+ 新对话、频道 +） |
